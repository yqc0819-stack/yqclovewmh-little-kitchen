# 我们的小厨房：从零开始的双人上线教程

适合：没有接触过 GitHub、数据库或网站部署的人。

预计时间：第一次操作约 60～90 分钟。不要赶，一步成功后再做下一步。

## 先说结论：需要付费吗？

夫妻两个人日常使用，通常可以全部使用免费方案：

| 服务 | 选择 | 费用 | 用来做什么 |
| --- | --- | --- | --- |
| GitHub | Free | 0 美元/月 | 保存网站代码 |
| Supabase | Free | 0 美元/月 | 保存菜单、照片、账号和实时消息 |
| Vercel | Hobby | 0 美元/月 | 把网站发布成手机能打开的网址 |
| 自定义域名 | 不购买 | 0 元 | 直接使用免费的 `xxx.vercel.app` 地址 |

按照 2026 年 7 月的官方说明：

- GitHub Free 支持无限公开或私有仓库。
- Supabase Free 包含 500MB 数据库、1GB 图片存储、每月 200 万条实时消息和 200 个同时连接。
- Vercel Hobby 是个人项目免费方案，没有付费账单周期。

两个人点餐远低于这些免费额度。

需要注意：

- Supabase 免费项目连续一周没有活动可能会暂停，再次进入后台可以恢复。
- 不要主动点击 Supabase 的 **Upgrade** 或 Vercel 的 **Upgrade to Pro**。
- 不买独立域名就不会产生域名费用。
- 如果未来大量上传原图，可能碰到 1GB 图片上限；本项目已经会自动压缩照片。

## 第 0 步：理解最后会得到什么

最后会有一个公网网站，例如：

```text
https://our-little-kitchen.vercel.app
```

两个人使用不同入口：

```text
做饭的人：
https://our-little-kitchen.vercel.app/?mode=admin

点餐的人：
https://our-little-kitchen.vercel.app/?mode=diner
```

操作流程：

```text
做饭人发布菜单
  → 菜单保存到 Supabase
  → 点餐人手机自动出现菜单
  → 点餐人提交选择
  → 做饭人手机出现提醒
  → 做饭人更新准备状态
  → 点餐人手机自动看到进度
```

## 第 1 步：注册 GitHub

GitHub 是放网站文件的地方。

1. 用电脑浏览器打开 <https://github.com/>。
2. 点击右上角 **Sign up**。
3. 输入常用邮箱。
4. 设置 GitHub 登录密码。
5. 设置用户名。建议使用英文和数字，例如 `xiaoming-kitchen`。
6. 完成人机验证。
7. 打开邮箱，点击 GitHub 发来的验证链接。
8. 回到 GitHub 登录。

成功标志：右上角能看到自己的圆形头像。

注意：GitHub 登录密码只在 GitHub 网站填写，不要写入项目文件，也不要发给别人。

---

## 第 2 步：在 GitHub 新建仓库

“仓库”可以理解为保存网站文件的文件夹。

1. 登录 GitHub。
2. 打开 <https://github.com/new>。
3. **Owner** 保持你的账号。
4. **Repository name** 填：

```text
our-little-kitchen
```

5. **Description** 可以填：

```text
我们的小厨房双人点餐网页
```

6. 可见性建议选择 **Private**，这样源代码默认只有你能看。
7. 不要勾选 **Add a README file**。
8. `.gitignore` 保持 **None**。
9. License 保持 **None**。
10. 点击绿色的 **Create repository**。

成功标志：进入一个空仓库页面，并看到 **Quick setup**。

---

## 第 3 步：把网站文件上传到 GitHub

这个方法不需要安装 Git，也不用敲命令。

1. 打开电脑目录：

```text
C:\Users\HUAWEI\Desktop\点餐小程序
```

2. 保持 GitHub 的仓库页面打开。
3. 在仓库页面点击 **uploading an existing file**。

如果没看到这个文字：

1. 点击 **Add file**。
2. 点击 **Upload files**。

4. 把“点餐小程序”文件夹里面的全部内容拖进上传区域。

应该包括：

```text
assets
supabase
index.html
styles.css
app.js
README.md
preview.ps1
双人在线部署教程.md
零基础双人上线教程.md
```

不要只上传 `index.html`，否则图片和功能会丢失。

5. 等待所有文件显示完成。
6. 在 **Commit changes** 的说明框填写：

```text
上传我们的小厨房第一版
```

7. 选择 **Commit directly to the main branch**。
8. 点击绿色的 **Commit changes**。

成功标志：

- 仓库文件列表中能看到 `index.html`、`styles.css` 和 `app.js`。
- 点击 `assets` 能看到 `anime-kitchen-hero.jpg`。
- 点击 `supabase` 能看到 `schema.sql`。

提醒：GitHub 网页单个文件上传上限是 25MB。本项目文件都低于这个限制。

---

## 第 4 步：注册 Supabase

Supabase 是保存两个人共享数据的地方。

1. 打开 <https://supabase.com/dashboard>。
2. 点击 **Start your project** 或 **Sign in**。
3. 建议选择 **Continue with GitHub**。
4. 页面跳转到 GitHub 时，点击授权。
5. 回到 Supabase。

成功标志：进入 Supabase Dashboard。

---

## 第 5 步：新建 Supabase 项目

1. 在 Supabase Dashboard 点击 **New project**。
2. 如果要求创建 Organization：
   - 名称可以填 `My Kitchen`。
   - Plan 选择 **Free**。
3. 回到新建项目页面。
4. **Name** 填：

```text
our-little-kitchen
```

5. **Database Password** 设置一个复杂密码。
6. 把这个数据库密码保存在自己的密码管理器或纸上。
7. **Region** 选择离你们较近的地区。
8. Plan 确认是 **Free**。
9. 点击 **Create new project**。
10. 等待项目创建，通常需要几分钟。

数据库密码不是网页的家庭登录密码。它只用于管理数据库，不要写进代码。

成功标志：进入项目首页，页面不再显示“正在创建项目”。

---

## 第 6 步：创建数据库表

项目中已经准备好了建表脚本，不需要自己设计表格。

1. 在 Supabase 左侧点击 **SQL Editor**。
2. 点击 **New query**。
3. 回到电脑项目目录。
4. 用记事本打开：

```text
supabase\schema.sql
```

5. 按 `Ctrl + A` 全选。
6. 按 `Ctrl + C` 复制。
7. 回到 Supabase SQL Editor。
8. 点击编辑区。
9. 按 `Ctrl + V` 粘贴。
10. 点击右下角或顶部的 **Run**。

成功标志：

- 页面显示 **Success**。
- 左侧进入 **Table Editor** 后能看到：
  - `households`
  - `profiles`
  - `dishes`
  - `daily_menus`
  - `menu_items`
  - `selections`
  - `selection_items`

如果显示红色错误：

- 先不要继续。
- 复制完整错误文字。
- 把错误发给 Codex 检查。
- 不要反复点击 Run，以免重复建立一半的数据。

---

## 第 7 步：创建做饭人账号

1. 在 Supabase 左侧点击 **Authentication**。
2. 点击 **Users**。
3. 点击 **Add user**。
4. 选择 **Create new user**。
5. Email 填一个你能识别的邮箱。

可以使用自己的邮箱别名，例如：

```text
yourname+cook@gmail.com
```

6. Password 设置至少 8 位。
7. 如果看到 **Auto Confirm User**，保持开启。
8. 点击 **Create user**。
9. 在用户列表找到刚创建的账号。
10. 复制它的 **User UID**，暂时保存到记事本，并标注“做饭人 UUID”。

成功标志：Users 列表中出现第一个账号。

---

## 第 8 步：创建点餐人账号

重复上一步，但使用另一个邮箱标识：

```text
yourname+diner@gmail.com
```

设置另一个至少 8 位的密码。

创建后复制 **User UID**，保存并标注“点餐人 UUID”。

建议：

- 两个账号可以使用不同密码。
- 密码只告诉对应使用者。
- 不要把密码放进 GitHub 或发给 Codex。

---

## 第 9 步：创建你们的家庭并绑定两个人

1. 回到 **SQL Editor**。
2. 点击 **New query**。
3. 打开本地文件：

```text
supabase\setup-family.sql
```

4. 先只复制文件最上面的第一段：

```sql
insert into public.households (name)
values ('我们的小厨房')
returning id;
```

5. 粘贴后点击 **Run**。
6. 下方 Results 会返回一个类似这样的 id：

```text
12345678-abcd-1234-abcd-1234567890ab
```

7. 复制它并标注“家庭 UUID”。
8. 新建一个 SQL Query。
9. 粘贴：

```sql
insert into public.profiles (user_id, household_id, role, nickname)
values
  ('做饭人用户UUID', '家庭UUID', 'cook', '今天掌勺的人'),
  ('点餐人用户UUID', '家庭UUID', 'diner', '今天等饭的人');
```

10. 替换三种中文占位内容：
    - `做饭人用户UUID`
    - `点餐人用户UUID`
    - 两处 `家庭UUID`
11. 保留每个 UUID 外面的英文单引号。
12. 点击 **Run**。

成功标志：显示 **Success. No rows returned**，并且 `profiles` 表中有两行数据。

---

## 第 10 步：取得前端连接信息

1. 在 Supabase 项目顶部点击 **Connect**。
2. 如果弹出的窗口标题是 **Connect to your project**，并且里面显示：

```text
Framework
Next.js
App Router
npm install @supabase/supabase-js
```

这是 Supabase 给各种开发框架准备的通用教程，不是报错。当前项目是纯 HTML/JavaScript，不需要安装 Next.js，也不要运行窗口中的 `npm install`。

3. 点击弹窗右上角的 `×` 关闭。
4. 点击 Supabase 左侧靠下的齿轮 **Settings** 或 **Project Settings**。
5. 点击 **API Keys**。
6. 找到 **Publishable key**：
   - 正确格式通常以 `sb_publishable_` 开头。
   - 点击右边的复制按钮。
   - 如果只有 **Create new API keys**，点击它创建默认密钥，然后只复制 Publishable key。
7. 在同一页面查看是否显示 **Project URL**。
8. 如果没有 Project URL，继续点击左侧 Settings 下的 **Data API**，复制页面中的 **Project URL**。

Project URL 看起来类似：

```text
https://abcdefghijk.supabase.co
```

Publishable key 看起来类似：

```text
sb_publishable_xxxxxxxxxxxxxxxxx
```

9. 在本地记事本中按下面格式保存：

```text
SUPABASE_URL=https://abcdefghijk.supabase.co
SUPABASE_PUBLISHABLE_KEY=sb_publishable_xxxxxxxxxxxxxxxxx
```

旧版项目有时只显示：

```text
anon public key
```

`anon public key` 是旧版的低权限浏览器密钥，也可以使用；新项目优先使用 `sb_publishable_` 开头的 Publishable key。

绝对不要复制或公开：

```text
Secret key（sb_secret_ 开头）
service_role
Database Password
JWT Secret
```

重要：做到这里，云端数据库已经准备好，但当前网页代码仍是本地存储版。

下一步需要把：

```text
Project URL
Publishable Key
做饭人登录邮箱
点餐人登录邮箱
```

配置到网页并把 `localStorage` 操作替换为 Supabase 请求。

你可以把 **Project URL** 和 **Publishable Key** 提供给 Codex继续接入；它们是浏览器端连接信息。不要提供两个账号的密码，密码应该只在最终网页登录框中输入。

## 第 10.5 步：运行云端升级脚本

如果 Codex 已经把 Project URL 和 Publishable Key 写入网页：

1. 回到 Supabase 的 **SQL Editor**。
2. 点击 **New query**。
3. 打开电脑项目中的：

```text
supabase\online-upgrade.sql
```

4. 全选复制到 SQL Editor。
5. 点击 **Run**。
6. 看到 **Success** 即完成。

这个脚本只补充浏览器访问权限和菜单明细实时同步，不会删除已有菜品或账号。

## 第 11 步：等待代码切换完成再部署

不要在代码还没连接 Supabase 时急着部署。

判断代码已经切换成功：

- 网页打开时出现家庭密码登录。
- 刷新网页后数据仍能从云端加载。
- 代码中能看到 Supabase 客户端初始化。
- `publishMenu()`、`submitChoice()` 和 `setStatus()` 不再只调用 `saveState()`。

如果直接部署当前本地版：

- 网站能打开；
- 但两台手机的数据依然不会互通。

Vercel 会自动提供 HTTPS，不需要单独购买证书。

## 常见问题

### 1. GitHub 上传后网页图片不显示

确认 `assets/anime-kitchen-hero.jpg` 已经上传，而且 `index.html` 位于仓库最外层。

### 2. Supabase SQL 运行报错

停止后续操作，复制完整红色错误信息进行检查。不要连续重复运行整份脚本。

### 3. Vercel 打开显示 404

确认仓库最外层存在 `index.html`，Root Directory 是 `./`。

### 4. 两台手机都能打开，但数据不同步

通常代表部署的仍是 `localStorage` 版本，还没有完成第 10～11 步的代码切换。

### 5. Supabase 项目暂停

免费项目长时间没有使用可能暂停。登录 Supabase Dashboard，进入项目并按照页面提示恢复。

### 6. 会不会突然扣费？

保持 Supabase Free 和 Vercel Hobby，不主动升级，通常不会产生月费。看到要求填写银行卡、升级 Pro 或购买域名时，可以先停止并检查自己是否误入了付费功能。

完成后保存好：

```text
Project URL
Publishable Key

点餐人邮箱
```

```text
任何密码
service_role Key

```

然后再进行代码联网和 Vercel 部署。
