# 我们的小厨房：双人在线互动与部署教程

> 如果你从未使用过 GitHub、Supabase 或 Vercel，请直接阅读更详细的 [零基础双人上线教程.md](./零基础双人上线教程.md)。该教程从注册账号和点击网页按钮开始。

这份教程的目标是让两台手机使用同一个网站：

- 做饭的人进入管理端，发布菜单、上传照片、更新进度。
- 点餐的人进入点餐端，选择菜品、填写备注。
- 任意一方操作后，另一台手机不用刷新就能看到变化。

## 一、它为什么现在不能跨手机同步

当前版本使用浏览器 `localStorage`。数据只存在打开网页的那台设备里，所以：

- 手机 A 新增的菜，手机 B 看不到。
- 复制本地文件地址没有意义。
- 清除浏览器数据后，记录会消失。

要解决它，需要增加三个云端部分：

```text
手机 A（管理端） ─┐
                  ├─ 网站（Vercel）
手机 B（点餐端） ─┘        │
                           ├─ 数据库（Supabase Postgres）
                           ├─ 菜品照片（Supabase Storage）
                           └─ 实时消息（Supabase Realtime）
```

## 二、最终使用方式

部署完成后会得到类似这样的固定网址：

```text
点餐端：https://your-kitchen.vercel.app/?mode=diner
管理端：https://your-kitchen.vercel.app/?mode=admin
```

两台手机分别收藏自己的链接。首次打开输入家庭密码，之后浏览器会保持登录。

## 三、需要准备的账号

1. Supabase 账号：保存数据、照片并负责实时同步。
2. GitHub 账号：保存网站代码。
3. Vercel 账号：把 GitHub 中的网站发布为公网网址。

Supabase 和 Vercel 的免费额度足够两个人日常使用。具体额度以后可能调整，请以各自官网为准。

## 四、创建 Supabase 项目

1. 打开 <https://supabase.com/dashboard>。
2. 点击 **New project**。
3. 填写项目名称，例如 `our-little-kitchen`。
4. 设置一个数据库密码并妥善保存。
5. Region 选择离两个人较近的地区。
6. 等待项目初始化完成。
7. 打开 **Project Settings → API** 或项目的 **Connect** 页面。
8. 保存：

```text
Project URL
Publishable Key
```

Publishable Key 本来就用于浏览器端，但必须配合 RLS 权限规则。绝对不要把 `service_role` Key 放进网页。

## 五、创建数据表和安全规则

1. 在 Supabase 左侧打开 **SQL Editor**。
2. 点击 **New query**。
3. 打开项目中的 [website/supabase/schema.sql](../website/supabase/schema.sql)。
4. 复制整份 SQL 到编辑器中。
5. 点击 **Run**。

该脚本会创建：

- 家庭与两位成员
- 菜品库
- 每日菜单
- 点餐结果与备注
- 菜品图片空间
- 两个人只能访问自己家庭数据的 RLS 规则
- Realtime 实时订阅

## 六、建立两个人的登录账号

为了保留“固定链接 + 简单密码”的体验，同时避免把密码写死在前端，推荐在 Supabase Auth 中建立两个固定账号。

1. 打开 **Authentication → Users**。
2. 创建做饭人账号，例如 `cook-你的代号@example.com`。
3. 创建点餐人账号，例如 `diner-你的代号@example.com`。
4. 给两个账号分别设置至少 8 位密码。
5. 复制两个用户的 UUID。
6. 回到 SQL Editor，执行：

```sql
insert into public.households (name)
values ('我们的小厨房')
returning id;
```

复制返回的家庭 UUID，再执行：

```sql
insert into public.profiles (user_id, household_id, role, nickname)
values
  ('做饭人用户UUID', '家庭UUID', 'cook', '今天掌勺的人'),
  ('点餐人用户UUID', '家庭UUID', 'diner', '今天等饭的人');
```

以后网页可以根据链接中的 `mode` 自动选择对应账号，只让使用者输入密码。

## 七、前端怎样连接 Supabase

当前代码还在使用 `localStorage`。正式接入时需要加入 Supabase JavaScript 客户端，并把这些函数替换为云端请求：

| 当前函数 | 接入后的作用 |
| --- | --- |
| `loadState()` | 查询菜品、今日菜单、选择结果和历史菜单 |
| `saveState()` | 不再整体保存，改成分别新增或更新数据库 |
| `handleDishImage()` | 压缩照片后上传到 `dish-images` |
| `publishMenu()` | 新建或更新 `daily_menus` 和 `menu_items` |
| `submitChoice()` | 写入 `selections` 和 `selection_items` |
| `setStatus()` | 更新 `daily_menus.status` |

初始化客户端：

```js
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

const supabase = createClient(
  "你的 Project URL",
  "你的 Publishable Key"
);
```

登录示意：

```js
await supabase.auth.signInWithPassword({
  email: mode === "admin" ? COOK_EMAIL : DINER_EMAIL,
  password: passwordInput
});
```

## 八、图片上传的同步流程

当前照片是 Base64 存在本机。接入后应改为：

1. 手机选择照片。
2. 浏览器继续使用现有代码压缩照片。
3. 上传到：

```text
dish-images/家庭UUID/随机文件名.jpg
```

4. 获取图片 URL。
5. 把 URL 保存到 `dishes.image_url`。
6. 另一台手机查询菜品时直接加载这个 URL。

这样照片不会占用手机浏览器的本地存储。

## 九、双人实时互动怎样发生

两台手机打开网页后，都订阅当前家庭的数据变化：

```js
supabase
  .channel("our-kitchen")
  .on(
    "postgres_changes",
    {
      event: "*",
      schema: "public",
      table: "daily_menus"
    },
    () => reloadTodayMenu()
  )
  .on(
    "postgres_changes",
    {
      event: "*",
      schema: "public",
      table: "selections"
    },
    () => {
      reloadSelection();
      showNotification();
    }
  )
  .subscribe();
```

实际流程：

```text
做饭人发布菜单
  → Supabase 写入今日菜单
  → 点餐人手机收到 Realtime 事件
  → 页面自动出现新菜单

点餐人提交选择
  → Supabase 写入选择结果
  → 做饭人手机收到 Realtime 事件
  → 红点、提示音或浏览器通知出现

做饭人点击“开始准备”
  → 状态更新为 preparing
  → 点餐人页面自动显示“开始准备啦”
```

浏览器通知需要用户首次点击“允许通知”。iPhone 上通常需要先把网页“添加到主屏幕”，再请求通知权限；页面内红点和提示条则不受这个限制。

## 十、部署到 GitHub

在项目目录初始化 Git：

```powershell
git init
git add .
git commit -m "我们的小厨房第一版"
```

在 GitHub 新建一个空仓库，然后按照 GitHub 页面给出的命令关联并推送：

```powershell
git branch -M main
git remote add origin 你的仓库地址
git push -u origin main
```

不要提交数据库密码或 `service_role` Key。

## 十一、部署到 Vercel

1. 打开 <https://vercel.com/new>。
2. 使用 GitHub 登录。
3. 导入刚创建的仓库。
4. Framework Preset 选择 **Other**。
5. 如果仍保持纯 HTML/CSS/JS，不需要 Build Command。
6. 点击 **Deploy**。
7. 等待完成后会得到 `https://项目名.vercel.app`。
8. 分别在两台手机打开：

```text
https://项目名.vercel.app/?mode=admin
https://项目名.vercel.app/?mode=diner
```

9. 添加到手机主屏幕或浏览器收藏夹。

以后每次把新代码推送到 GitHub 的 `main` 分支，Vercel 会自动重新部署。

## 十二、上线前必须检查

- 两台手机使用不同账号登录。
- 手机 A 发布菜单后，手机 B 不刷新也能看到。
- 手机 B 提交后，手机 A 能出现红点提醒。
- 菜品图片在两台手机上都能打开。
- 点餐账号无法进入菜品管理操作。
- 管理账号可以更新“已看到 / 准备中 / 已做好”。
- Supabase 表全部启用了 RLS。
- 网页代码中不存在 `service_role` Key。

当前已准备好：

- 页面和动漫视觉
- 本地图片上传与压缩
- 数据库建表及 RLS 脚本
