# 我们的小厨房：云端接入说明

> 本文已升级为更详细的 [双人在线部署教程.md](./双人在线部署教程.md)，包含账号创建、数据库脚本、实时互动、图片同步、GitHub 与 Vercel 部署的完整步骤。以下内容保留为方案摘要。

目前网页把菜品、菜单和照片保存在浏览器 `localStorage`，所以只能在同一浏览器中演示。要让两台手机通过同一个链接实时同步，推荐使用：

- Supabase：Postgres 数据库、菜品图片 Storage、Realtime 状态同步
- Vercel：部署当前 HTML/CSS/JS，生成家庭固定网址

## 接入顺序

1. 在 Supabase 新建项目。
2. 建立 `households`、`dishes`、`daily_menus`、`menu_items`、`selections`、`selection_items` 六张表。
3. 新建 `dish-images` Storage bucket，用于保存菜品照片；数据库只保存图片 URL。
4. 为菜单、选择结果和进度表开启 Realtime。
5. 前端通过 Project URL 和 Publishable Key 创建 Supabase 客户端。
6. 把当前项目放入 Git 仓库，再导入 Vercel 部署。

## 建议的数据关系

```text
households
  ├─ dishes
  ├─ daily_menus
  │    └─ menu_items
  └─ selections
       └─ selection_items
```

每条数据都带 `household_id`。点餐链接使用不可猜测的家庭 ID；管理操作还需要单独的管理员口令。口令不能明文写在前端代码里，应在服务端校验或使用 Supabase Auth。

## 前端需要替换的部分

- `loadState()`：改成从 Supabase 查询家庭数据。
- `saveState()`：拆成对应数据表的新增或更新请求。
- `handleDishImage()`：压缩后上传到 Storage，并保存返回的图片 URL。
- `submitChoice()`：写入 `selections` 与 `selection_items`。
- `setStatus()`：更新选择结果的状态。
- 页面初始化：订阅菜单、选择结果和状态表的 Realtime 变化。

真正接入时需要准备两个值：

```text
SUPABASE_URL
SUPABASE_PUBLISHABLE_KEY
```

Publishable Key 可以用于浏览器端；不要把 `service_role` 密钥放进网页。
